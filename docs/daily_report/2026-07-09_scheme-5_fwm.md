## 현재 정의와 물리 구현

### 1. Seeded FWM / squeezing

Seeded 모드는 `85Rb D1` double-Lambda four-wave mixing을 compact 4-level Maxwell-Bloch 모델로 계산한다. ground/excited hyperfine line strength는 `TRANSITION_DIPOLE_SCALE[g,e]^2 = 3 * C_F^2`로 들어가며 (`gabes/schemes/fwm.py:35-40`), macroscopic coupling에는 validated absorption 경로와 맞추기 위한 `p_F/[2(2I+1)]` 정규화가 별도로 들어간다 (`gabes/schemes/fwm.py:43-62`). 현재 `line_strength`는 이 물리 정규화 위에 얹는 residual calibration으로 문서화되어 있다 (`gabes/schemes/fwm.py:1462-1468`, `gabes/schemes/fwm.py:1801-1810`).

실험물리학자 관점에서 이 선택은 좋다. `line_strength`가 더 이상 "실험자가 실제로 돌리는 기본 knob"처럼 보이지 않고, compact model이 흡수하지 못한 beam overlap, polarization purity, participating Zeeman fraction 같은 잔여 효과를 대표한다는 해석이 가능하기 때문이다. 또 `tests/test_regression.py:75-83`은 hyperfine strength와 branching이 실제 85Rb D1 값과 맞는지 고정하고, `tests/test_regression.py:52-62`는 서로 다른 Raman branch를 한 susceptibility로 합치는 오류를 막는다.

다만 reference 수준은 "semi-quantitative experiment steering"에 가깝다. `pump_depletion_saturation()`은 Manley-Rowe energy bound를 걸어 high-density runaway를 누르지만 (`gabes/observables.py:87-115`), 이것은 full propagation + quantum Langevin noise가 아니라 small-signal gain 위에 얹는 포화 guardrail이다. Ultra fidelity에는 phase matching, segmented propagation, dynamic depletion, in-cell loss/noise proxy가 붙지만 (`gabes/schemes/fwm.py:1712-1718`, `gabes/schemes/fwm.py:2138-2148`), full 24-level Zeeman Floquet solve는 아직 main solve가 아니라 future work로 남아 있다 (`README.md:216-218`, `docs/checklist.json:98-101`).

따라서 seeded FWM은 실제 85Rb D1 warm-vapor squeezing 실험의 detuning, temperature, gain/squeezing trend, loss sensitivity, branch 해석을 잡는 데는 충분히 유용하다. 절대 gain과 최종 noise budget을 논문 수치처럼 인용할 때는 residual calibration과 noise-model 한계를 같이 인용해야 한다.

### 2. Generic SFWM / biphoton

Biphoton 모드는 같은 스킴 안에서 `87Rb telecom cascade`, `133Cs BTW cascade`, `generic diamond` topology를 다룬다 (`gabes/schemes/fwm.py:1753-1760`). 기본 source model은 `Predictive`이고, Doppler-averaged cascade amplitude, `Omega_c^2` Autler-Townes denominator, velocity-class coherent sum, natural-linewidth decay를 사용한다 (`gabes/schemes/fwm.py:710-771`). 특히 `_converged_predictive_waveform()`은 velocity-space source resonance를 oversample하고 FWHM이 안정될 때까지 grid를 반으로 줄인다 (`gabes/schemes/fwm.py:829-871`). 이는 6월 24일 보고서에서 지적된 coarse velocity-step aliasing을 현재 코드가 상당히 해소했음을 뜻한다.

Rate는 여전히 reference-anchored scaling이다. 코드도 pair-rate magnitude가 lumped 4-level model에서 절대 collection coefficient를 pin하지 못하기 때문에 reference anchor를 쓴다고 명시한다 (`gabes/schemes/fwm.py:921-931`). 즉 waveform shape, phase-matching sensitivity, detector/background trade-off는 물리적으로 읽을 만하지만, absolute pair rate를 독립 예측값처럼 쓰면 안 된다.

Phase matching 쪽은 실험적으로 특히 유용하다. longitudinal mismatch뿐 아니라 transverse vector mismatch를 계산하고, Fine 모드에서는 `(121, 121)` 2D acceptance map까지 만든다 (`gabes/schemes/fwm.py:985-1018`). `tests/test_fwm_generic.py:65-120`은 reference geometry가 최대 rate를 주고, 같은 각도/잘못된 side 선택이 rate를 강하게 억제하는지 검증한다. 실제 collection geometry를 맞추는 실험자에게 이 부분은 단순 toy보다 훨씬 쓸모 있다.

## 이전 개선안과 부하 평가

### 이미 반영된 개선: figureless observables

`figureless-observables-paths`는 `done` 상태다 (`docs/checklist.json:28-31`). FWM도 `supports_headless_observables = True`이고 (`gabes/schemes/fwm.py:1736-1737`), `observables(..., include_figures=False)`가 seeded/biphoton 양쪽을 통과한다 (`gabes/schemes/fwm.py:2073-2081`). 오늘 timing에서도 headless path는 거의 0초였다. 이는 자동 보고, batch metric extraction, CI에서 기능 손상 없이 큰 체감 이득을 준다.

### `fwm-per-source-biphoton-calibration`

이 개선안은 현재 가장 좋은 물리 대비 비용 trade-off다 (`docs/checklist.json:42-45`). topology별 effective linewidth, dephasing, coupling scale anchor를 분리하면 predictive width를 "한 lumped approximation"에서 "source별 calibrated prediction"으로 올릴 수 있다. solver dimension은 늘리지 않고 scalar/low-dimensional parameters만 추가하므로 연산 부하는 사실상 미미하다. UI/docs/tests는 필요하지만, runtime 비용은 현재 predictive solve와 거의 같을 것이다.

### `fwm-line-strength-factorization`

현재 residual `line_strength`를 beam overlap, polarization purity, effective participating Zeeman fraction 등 lab-facing factors로 분해하는 제안이다 (`docs/checklist.json:49-52`). 이것 역시 solver 차원 증가 없이 곱셈 factor를 구조화하는 작업이라 계산 부하는 거의 없다. 물리적으로는 "0.74를 왜 넣었는가"를 실험가 언어로 설명할 수 있게 해 준다. 단점은 각 factor의 default/범위가 새 calibration 책임을 만든다는 점이다.

### `fwm-batch-cache-and-map-separation`

순수 성능 개선으로는 이 항목이 가장 직접적이다 (`docs/checklist.json:70-73`). 현재 topology solver는 `_solver_from_topology_key(_topology_cache_key(params))`로 캐시되는 경로가 있고 (`gabes/schemes/fwm.py:2049-2051`), 2D phase map은 Fine일 때만 계산된다 (`gabes/schemes/fwm.py:985-993`). 이미 꽤 잘 분리되어 있지만, large batch scan에서는 velocity grid, scan axis, acceptance map 같은 불변 배열을 더 적극적으로 재사용할 여지가 있다. 물리 기능을 바꾸지 않는 최적화이며, 특히 많은 parameter sweep에서만 이득이 크다.

### GROUP C: quantum-Langevin noise, full Zeeman Floquet

`fwm-quantum-langevin-noise`와 `fwm-full-zeeman-floquet-solve`는 모두 GROUP C다 (`docs/checklist.json:91-101`). 이들은 물리적으로 매력적이지만 계산 부하와 검증 면적이 크다. Langevin noise는 현재 waveform + detector accidentals를 넘어 source noise propagation 상태를 추가해야 하고, full Zeeman Floquet는 4-level compact solve를 24-level density-matrix/Floquet 문제로 키운다. 실험물리학적 reference 등급을 올리는 핵심 방향이지만, "부하를 거의 만들지 않으면서 물리를 따라가는 개선"으로 보기는 어렵다.

## 순수 코딩 관점의 최적화 후보

1. Batch scan에서는 `headless_observables()`를 표준 경로로 사용한다. 이미 구현되어 있고, 오늘 측정에서 figure build 비용을 거의 제거했다 (`gabes/schemes/fwm.py:2073-2081`, `tests/test_fwm_generic.py:242-260`).
2. Fine phase 2D map은 UI/extra view 중심으로 더 늦게 계산할 수 있다. 현재도 opt-in이지만, 자동 batch에서 accidental Fine selection이 섞이면 `(121,121)` map이 반복 생성된다 (`gabes/schemes/fwm.py:985-993`).
3. Seeded scan의 velocity grid와 scan axis는 fidelity/temperature 조합별로 재사용 여지가 있다 (`gabes/schemes/fwm.py:1712-1718`, `gabes/schemes/fwm.py:1490-1493`). 물리는 바꾸지 않고 repeated sweeps에서 allocation과 setup 비용을 줄일 수 있다.
4. `line_strength` factorization이나 per-source biphoton calibration은 물리 개선이지만 계산 최적화와도 잘 맞는다. 둘 다 high-dimensional solve를 추가하지 않고 scalar bookkeeping을 더 명확히 하는 방향이다.

## 종합 평가

오늘 기준 `FWMScheme`은 GABES에서 가장 실험 reference에 가까운 스킴 중 하나다. Seeded FWM은 85Rb D1 hyperfine strength, branch 분리, Sim et al. anchor, loss/squeezing readout이 잘 정리되어 있어 실제 warm-vapor squeezing 실험의 operating-point 탐색과 sanity check에 충분히 유용하다. 다만 absolute source metrology로 쓰려면 residual calibration, pump-depletion guardrail, 아직 없는 full quantum-Langevin/Zeeman solve 한계를 반드시 함께 적어야 한다.

Biphoton 모드는 더 설계 도구에 가깝다. Geometry, vector phase matching, detector background, coincidence window, timing jitter를 한 자리에서 다루는 점은 실험 준비에 실제 가치가 크다. Predictive waveform은 이제 velocity aliasing 문제를 상당히 벗어났고 RB87 telecom width도 reference scale에 가깝게 수렴하지만, absolute pair rate와 topology별 exact width는 여전히 calibration 성격을 가진다.

다음으로 가장 부담이 작고 물리적 이득이 큰 개선은 `fwm-per-source-biphoton-calibration`과 `fwm-line-strength-factorization`이다. 둘 다 solver 차원을 늘리지 않고 실험가가 이해할 수 있는 calibration 언어를 제공한다. 순수 속도 측면에서는 이미 구현된 headless path를 batch/report에서 적극 사용하고, Fine phase map과 반복 grid 생성을 더 엄격히 lazy/cache 처리하는 것이 가장 안전하다.
