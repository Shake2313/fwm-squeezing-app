## 현재 정의와 구조

### 1. seeded FWM / squeezing 모드

- 이 모드는 `85Rb D1 double-Lambda` 4준위 seeded FWM을 유지한다. UI 캡션도 “legacy seeded gain model”을 명시한다 (`gabes/schemes/fwm.py:1452-1454`).
- 계산 본체 `compute_spectrum()`은 one-photon detuning `opd`, 온도, 셀 길이, pump/seed 파워, line-strength residual을 받아 gain 및 squeezing 곡선을 만든다 (`gabes/schemes/fwm.py:1204-1245`).
- branch를 합산하지 않고 한 Raman branch씩 계산하도록 강제한다. 이는 실험적으로도 맞는 선택이다. 서로 다른 branch는 같은 susceptibility에 더할 대상이 아니라 서로 다른 probe/conjugate mode pair이기 때문이다 (`gabes/schemes/fwm.py:1189-1194`, `tests/test_regression.py:52-62`, `README.md:166-169`).
- `line_strength`는 실험자가 직접 조절하는 물리 knob라기보다, 계산되지 않은 거시적 결합/기하 감쇠를 흡수하는 residual calibration으로 분리되어 있다 (`gabes/schemes/fwm.py:1515-1527`, `gabes/schemes/fwm.py:1218-1224`, `docs/FWM_physics.tex:1017-1020`).

### 2. generic SFWM / biphoton 모드

- 같은 스킴 안에서 `mode = Squeezing / Biphoton`으로 갈라진다 (`gabes/schemes/fwm.py:1462-1469`, `gabes/schemes/fwm.py:1750-1754`).
- biphoton 모드는 `87Rb telecom cascade`, `133Cs BTW cascade`, `generic diamond` 세 topology를 제공한다 (`gabes/schemes/fwm.py:1467-1479`, `docs/FWM_physics.tex:868-879`).
- 현재 기본 source model은 `Predictive`이며, Doppler 평균된 cascade biphoton amplitude, velocity-class coherent sum, natural-linewidth decay, vector phase matching을 포함한다 (`gabes/schemes/fwm.py:1475-1487`, `gabes/schemes/fwm.py:644-725`, `gabes/schemes/fwm.py:727-885`).
- 다만 absolute pair rate와 일부 width는 여전히 anchor를 남겨 둔다. 코드와 문서 모두 그 한계를 숨기지 않는다 (`gabes/schemes/fwm.py:1715-1732`, `gabes/schemes/fwm.py:1955-1980`, `README.md:133-143`, `docs/checklist.json:12-16`).

## 로컬 실행 결과

오늘 로컬에서 다음을 실행해 확인했다.

- `python tests/test_regression.py`
- `python tests/test_fwm_generic.py`

결과:

- 두 테스트 모두 통과했다.
- `tests/test_regression.py`는 refactor 이후 FWM이 frozen baseline을 bit-level로 유지하는지 검증한다 (`tests/test_regression.py:42-49`).
- `tests/test_fwm_generic.py`는 위상정합 geometry, detector background, timing jitter, predictive width ordering, Zeeman lumped-strength consistency 등을 폭넓게 본다 (`tests/test_fwm_generic.py:65-245`, `tests/test_fwm_generic.py:360-421`).

추가로 오늘 직접 대표 계산을 돌린 결과는 다음과 같았다.

- seeded 기본값(`121 C`, `600 mW`, `8 uW`)에서 `min S_dB ~= -8.44 dB`, `max G_s ~= 6.99`였다.
- 같은 seeded 조건에서 `110 C -> 121 C -> 130 C`로 올리면 `max G_s ~= 1.65 -> 6.99 -> 105.87`로 매우 급격히 커졌다. README가 경고한 “고밀도에서 gain의 지수 민감성”과 잘 맞는다 (`README.md:161-166`).
- biphoton `87Rb telecom` preset은 calibrated 모드에서 `FWHM ~= 0.54 ns`, predictive 모드에서 `FWHM ~= 4.85 ns`가 나왔다. 즉 정성적 waveform physics는 살아 있지만, 절대 ns 폭은 predictive에서 아직 reference급으로 믿기 어렵다는 코드의 자기평가와 일치했다 (`README.md:133-143`, `docs/checklist.json:15`, `gabes/schemes/fwm.py:1961-1970`).
- biphoton `Cs BTW` predictive에서는 `852-917 nm` 채널이 `852-795 nm`보다 더 좁게 나왔고, 이는 테스트 의도와 동일하다 (`tests/test_fwm_generic.py:200-213`).
- `87Rb telecom` biphoton에서 일부러 잘못된 geometry(`1.5 deg / 1.5 deg`, same side)를 주면 pair rate가 기준의 약 `2.4e-6` 수준으로 붕괴했다. 이는 vector phase matching을 넣어 둔 구현이 실제로 작동한다는 뜻이다 (`tests/test_fwm_generic.py:102-120`, `docs/FWM_physics.tex:888-928`).

## 실험물리학 연구자의 관점에서 본 평가

### seeded FWM는 얼마나 유용한가

- 이 모드는 단순 toy model보다 훨씬 낫다. 실제 `85Rb D1` hyperfine CG 가중치와 branching을 반영하고 (`tests/test_regression.py:65-83`, `docs/FWM_physics.tex:1013-1015`), 실험 논문 Sim et al.의 동작점을 regression anchor로 고정해 두었다 (`gabes/schemes/fwm.py:1522-1526`, `docs/FWM_physics.tex:1009-1013`).
- 또한 probe/conjugate gain과 squeezing을 함께 읽게 해 주므로, “이 온도와 detuning에서 gain만 큰지, 실제 IDS squeezing까지 유지되는지”를 같은 그림에서 판단하기 좋다 (`gabes/observables.py:248-280`, `gabes/schemes/fwm.py:1782-1882`).
- 따라서 **seeded FWM는 warm-vapor 85Rb D1 twin-beam 실험에서 동작점 탐색, 파라미터 감도 스캔, branch 해석, loss impact 감각을 잡는 데 충분히 유용한 반정량 레퍼런스**라고 본다.

하지만 동시에 한계도 분명하다.

- 고온에서 gain이 지나치게 빨리 커지는 것은 Maxwell-Bloch small-signal propagation의 본질적 경향이고, 코드도 이를 감추지 않고 pump-depletion cap을 별도로 넣는다 (`gabes/observables.py:87-115`, `gabes/schemes/fwm.py:1704-1714`).
- full Zeeman Floquet solve는 아직 하지 않고, 24-level manifold는 diagnostic correction 용도다 (`README.md:216-218`, `docs/FWM_physics.tex:1022-1034`).
- 즉 **정량 피팅의 최종 기준기**라기보다는, “어느 쪽으로 실험을 움직이면 gain/squeezing이 좋아지는가”를 빠르게 보는 compact 실험 설계 도구에 가깝다.

### biphoton 모드는 얼마나 유용한가

- 이 부분은 흥미롭지만 성격이 다르다. geometry, topology, detector background, coincidence window, timing jitter까지 한 자리에서 보게 해 주는 점은 실제 실험 준비에 매우 유용하다 (`gabes/schemes/fwm.py:1885-2055`, `gabes/observables.py:174-245`).
- 특히 phase matching을 longitudinal calibration만이 아니라 transverse vector mismatch까지 포함해 suppression시키는 점은 실험가에게 실질적인 가치가 크다 (`gabes/schemes/fwm.py:742-764`, `docs/FWM_physics.tex:888-928`).
- predictive 모드가 wavelength-dependent width ordering, coupling-driven bandwidth 변화 같은 정성 physics를 살려 두었다는 것도 좋다 (`tests/test_fwm_generic.py:200-228`, `gabes/schemes/fwm.py:644-725`).

다만 **절대 시간폭과 절대 pair-rate를 곧바로 논문급 숫자로 믿기에는 아직 이르다**.

- 코드가 스스로 말하듯 pair rate는 anchored scaling이고 (`gabes/schemes/fwm.py:775-785`, `gabes/schemes/fwm.py:1958-1960`), predictive absolute width도 topology에 따라 과민할 수 있다 (`docs/checklist.json:15`, `README.md:133-143`).
- 따라서 biphoton 모드는 오늘 기준으로 **실험 geometry/검출 조건/상대 경향을 읽는 설계 참고서**로는 좋지만, absolute source metrology reference로는 조심해서 써야 한다.

## 저장소에 이미 있던 개선안과 계산 부하

이번 검토에서 FWM에 직접 연결되는 기존 개선안은 명시적으로 하나 찾았다.

- `biphoton-predictive-model` (`docs/checklist.json:12-16`)

이 항목은 이미 “부분 완료” 상태이며, 남은 숙제는 크게 둘로 나뉜다.

### 1. per-source Rabi / dephasing calibration

- checklist가 말하는 “per-source calibration for exact widths”는 현재 `Predictive` 파이프라인의 구조를 유지한 채 topology별 effective linewidth, dephasing, coupling scale anchor를 다듬는 일에 가깝다 (`docs/checklist.json:15`, `gabes/schemes/fwm.py:581-605`, `gabes/schemes/fwm.py:660-713`).
- 이 방향은 계산 부하가 거의 늘지 않는다. 이유는 solve 차원이 커지는 것이 아니라, 이미 있는 scalar/low-dimensional parameter를 topology별로 재보정하는 수준이기 때문이다.
- 즉 **물리를 더 따라가면서도 연산 부하는 사실상 거의 유지할 수 있는 개선안**이다.

### 2. Langevin noise channel

- full quantum-Langevin noise를 넣으려면 현재의 “waveform + detector accidentals” readout을 넘어 source noise 자체를 동역학적으로 다뤄야 한다 (`docs/checklist.json:15`, `README.md:141-143`, `docs/FWM_physics.tex:1017-1034`).
- 이건 단순 post-processing이 아니라 observables 층뿐 아니라 source model의 정의 자체를 바꾸는 일이다.
- 구현 방식에 따라 다르지만, 최소한 correlation/noise propagation 차원이 올라가고, 보통은 지금보다 계산·검증 부담이 커질 가능성이 높다.
- 따라서 **실험물리적 가치는 크지만 “부하가 거의 없는 개선” 범주로 보기는 어렵다**.

정리하면, 저장소에 이미 적혀 있는 FWM 개선안 중 당장 우선순위가 높은 것은 **topology별 width/dephasing/Rabi calibration**이고, 이건 계산 속도를 거의 해치지 않으면서 물리를 가장 많이 끌어올릴 수 있다.

## 추가로 조심스럽게 제안할 수 있는 물리 개선 후보

기존 메모를 넘어, 코드 구조를 보며 조심스럽게 제안할 수 있는 후보는 다음 정도다.

### 1. seeded FWM의 `line_strength`를 더 물리적인 하위 인자들로 분해

- 현재 `line_strength`는 실험적으로는 매우 유용하지만, 여러 효과를 한 숫자에 모아 둔 residual이다 (`gabes/schemes/fwm.py:1515-1527`, `gabes/schemes/fwm.py:1218-1224`).
- beam overlap, polarization purity, effective participating Zeeman fraction 같은 하위 요인으로 분해하면, absolute gain discrepancy를 해석할 때 실험가에게 더 직접적인 언어가 된다.
- 이 역시 full 24-level solve 없이도 phenomenological factorization만으로 시작할 수 있어서 계산 부하는 작다.

### 2. predictive biphoton의 topology별 absolute width anchor 분리

- 현재 문서가 인정하듯 telecom case와 Cs BTW case의 신뢰 수준이 다르다 (`README.md:133-143`, `docs/checklist.json:15`).
- 그러므로 topology마다 다른 effective dephasing / lifetime correction을 두고 validation table에서 “정성 예측”과 “정량 anchor”를 더 명확히 분리하는 편이 실험 참고서로서 정직하다.
- 이것도 부하 증가는 거의 없다.

## 순수 코딩 측면의 속도 개선 후보

물리를 바꾸지 않으면서 속도를 올릴 수 있는 부분도 보인다.

### 1. biphoton solver 객체 재사용 / topology 캐시

- 현재 biphoton compute는 호출마다 `GenericFWMSolver(topology)`를 새로 만든다 (`gabes/schemes/fwm.py:1751-1754`).
- 객체 자체는 가볍지만, topology별 불변 파생량을 `lru_cache`나 scheme-level cache로 묶으면 배치 스캔에서 미세한 이득이 있다.
- 기대 이득은 크지 않지만, 물리 침해는 없다.

### 2. seeded observables에서 figure 생성과 metric 계산 분리

- 지금은 `observables()`가 Matplotlib figure를 항상 같이 만든다 (`gabes/schemes/fwm.py:1782-1882`, `gabes/schemes/fwm.py:1885-2055`).
- 테스트/배치 분석/자동화 보고에서는 metric과 table만 필요할 때가 많다.
- figure-less path를 두면 자동화나 CI에서 체감 속도를 줄일 수 있다. 물리는 그대로이고 UI만 경로 분기하면 된다.

### 3. predictive biphoton의 angle acceptance map 계산 조건 더 엄격히 분리

- `phase_detail = Fine`일 때 2D phase-matching map을 추가로 계산한다 (`gabes/schemes/fwm.py:840-848`, `tests/test_fwm_generic.py:368-386`).
- 현재도 opt-in이지만, large batch에서는 이 부분이 불필요한 후처리가 될 수 있다.
- 계산 결과 캐시 키를 조금 더 세밀하게 나누거나, map 계산을 extra view로 빼면 physics는 그대로 두고 UI 반응성을 더 올릴 수 있다.

### 4. seeded 고충실도 스캔에서 fidelity별 grid 캐시 강화

- fidelity에 따라 `coarse_points`, `velocity_step`, `velocity_cutoff`, `phase_detail`가 정해진다 (`gabes/schemes/fwm.py:1426-1433`).
- 이 값 조합이 반복될 때 scan axis와 velocity grid 같은 불변 배열을 재사용하면, 여러 detuning/temperature sweep 자동화에서 도움이 된다.
- 이 역시 물리 변경 없이 가능한 최적화다.

## 종합 판단

- 오늘 기준 `FWMScheme`의 seeded 모드는 **실제 85Rb D1 warm-vapor FWM 실험을 준비하거나 해석하는 연구자에게 충분히 쓸모 있는 반정량적 레퍼런스**다. 특히 branch 분리, hyperfine strength, Sim et al. anchor, loss/squeezing readout이 잘 정리돼 있다.
- biphoton 모드는 **실험 geometry와 detector 조건을 포함한 설계 도구**로는 상당히 유용하지만, absolute rate/width 예측은 아직 calibration 의존성이 남아 있어서 reference-grade metrology tool로 보기에는 조심해야 한다.
- 기존 개선안 중 최우선은 `per-source Rabi/dephasing calibration`이며, 이것은 **연산 부하를 거의 늘리지 않으면서 물리적 신뢰도를 가장 효율적으로 높일 수 있는 방향**이다.
