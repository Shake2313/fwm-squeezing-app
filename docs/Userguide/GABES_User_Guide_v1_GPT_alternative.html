GABES
Generic Atomic Bloch Equation Solver

GABES는 따뜻한 원자 증기와 냉각 원자 분광에서 자주 만나는 Optical Bloch Equation을 빠르게 탐색하기 위한 실험 친화적 toy-model solver입니다. “toy”라고 부르지만, 실제 원자광학 사용자가 파라미터 감각을 잡을 수 있도록 Doppler 평균, hyperfine structure, Zeeman manifold, Λ coherence, FWM gain/squeezing 등을 꽤 깊게 포함합니다.

1. GABES란?

GABES의 목적은 “논문급 full simulation”을 대체하는 것이 아니라, 실험자가 OBE 기반 현상을 빠르게 훑고, 파라미터 변화에 대한 감을 얻고, 다음 실험 조건을 정하는 데 도움을 주는 것입니다.

핵심 문장: GABES는 Optical Bloch Equation의 toy model solver입니다. 다만 단순한 2-level 장난감에 머무르지 않고, 실제 알칼리 원자 분광에서 중요한 Doppler 평균, hyperfine pumping, Λ coherence, Zeeman coherence, FWM propagation 같은 요소를 가능한 한 계산 가능한 형태로 끌어들입니다.

잘 맞는 용도

  • OD/SAS, EIT/AT/CPT, Hanle/NMOR, Rydberg-EIT, FWM 같은 현상의 파라미터 의존성 탐색
  • 온도, cell length, detuning, Rabi frequency, magnetic field scale 변화에 대한 1차 예측
  • 실험 데이터와 비교하기 전, “대충 어디를 봐야 하는지” 정하는 지도 제작
  • 새 scheme을 추가하기 위한 공통 OBE 계산 골격

조심해야 할 용도

  • 모든 세부 실험 요소를 포함한 최종 fitting engine으로 쓰는 것
  • absolute noise, pump scattering, residual sideband, window birefringence 같은 실험 잡음을 자동으로 설명한다고 기대하는 것
  • 모델 범위를 벗어난 조건에서 수치 결과를 정량적으로 과신하는 것

2. 빠른 시작

로컬에서 실행할 때는 저장소 루트에서 아래 명령을 사용합니다. 

pip install -r requirements.txt
streamlit run streamlit_app.py
주의: python streamlit_app.py로 실행하는 방식이 아닙니다. Streamlit 서버를 띄우는 앱이므로 streamlit run streamlit_app.py를 사용합니다.

FWM backend를 명령행에서 직접 확인하고 싶을 때는 호환용 entry point인 python fwm_obe.py를 사용할 수 있습니다. 보통 사용자는 Streamlit UI를 권장합니다.

3. 화면 사용법

GABES의 화면은 “scheme 선택 → 파라미터 조정 → 계산 결과 확인”으로 구성됩니다. UI는 고정된 슬라이더 벽이 아니라, 선택한 물리 scheme이 필요한 컨트롤만 선언하면 앱이 그 컨트롤과 plot을 자동으로 그리는 방식입니다.

1. Scheme 선택좌측 sidebar에서 실험/물리 구도를 고릅니다.
2. Preset 적용가능한 경우 기본 조건을 먼저 불러옵니다.
3. 주요 knob 조정detuning, 온도, power, B scan 등을 조정합니다.
4. Metric 확인gain, squeezing, OD, linewidth, rotation 등 핵심 숫자를 봅니다.
5. Table/extra viewderived quantity와 느린 확장 계산은 expander에서 확인합니다.

Recompute knob과 navigate knob

일부 knob은 값을 바꾸면 OBE를 다시 풉니다. 다른 knob은 이미 계산된 곡선에서 읽는 위치만 옮기므로 즉시 반응합니다. 예를 들어 seeded FWM에서 OPD, 온도, pump power는 재계산 대상이지만, TPD readout은 캐시된 곡선 위의 operating point를 옮기는 동작입니다.

Advanced / numerics

고급 수치 설정은 기본적으로 접혀 있습니다. velocity step, scan points, Doppler on/off, transit relaxation 같은 항목은 결과를 바꿀 수 있지만, 처음부터 만질 필요는 없습니다. 먼저 preset과 기본 knob로 모양을 본 뒤, 필요할 때만 펼쳐서 조정하는 것을 권합니다.

4. Scheme별 기능

현재 GABES는 크게 absorption, magneto-optics, wave mixing 계열로 나뉩니다. 각 scheme은 자체 파라미터, 계산 루틴, metric, plot, table을 가집니다.

Scheme 주요 물리 주요 출력 먼저 만질 knob
OD / SAS weak-probe absorption, Doppler-broadened OD, saturated absorption, hyperfine optical pumping Transmission, OD, Doppler FWHM, hyperfine line table Pump power, species/isotope, D1/D2, temperature, cell length
Λ coherence
EIT / AT / CPT		 3-level Λ system, transparency, Autler-Townes splitting, dark resonance Transmission, Re χ, window FWHM, group index, AT splitting Regime, coupling Rabi, ground dephasing, Doppler, temperature
Rydberg-EIT electrometry 85Rb 5S-5P-40D cascade EIT, 37 GHz microwave AT dressing EIT linewidth, RF AT splitting, optical dispersion EIT/AT mode, coupling Rabi, microwave LO Rabi, microwave detuning
Hanle / EIA / NMOR 87Rb D1 Zeeman manifold, near-zero-field transmission and magneto-optical rotation Hanle feature, NMOR rotation, B-field width, derived cell/relaxation quantities Signal type, cell type, transition, intensity, QWP angle, B scan range
FWM
Squeezing / Biphoton		 85Rb D1 double-Λ seeded FWM gain/squeezing, generic SFWM biphoton estimates Probe gain, conjugate gain, intensity-difference squeezing, g²SI, CAR, pair rate Mode, OPD, TPD, temperature, pump/probe power, phase-matching angle

A OD / SAS

Pump power를 0으로 내리면 linear Doppler-broadened OD가 됩니다. Pump를 올리면 velocity-selective saturation과 hyperfine optical pumping으로 Lamb dip과 crossover가 나타납니다. Natural Rb, 85Rb, 87Rb, Cs, D1/D2 선택을 지원합니다.

Λ EIT / AT / CPT

하나의 Λ engine을 EIT, AT, CPT 세 regime으로 보여줍니다. coupling field가 약하면 transparency window, 강하면 AT doublet, 좁은 two-photon scan에서는 CPT dark resonance를 봅니다.

B Magneto-optics

87Rb D1 Zeeman manifold에서 zero-field 근처의 Hanle transmission과 NMOR rotation을 계산합니다. QWP angle, residual transverse field, cell relaxation이 EIT-like dip 또는 EIA/MIA-like peak를 바꿉니다.

D FWM

Squeezing mode는 85Rb D1 double-Λ seeded FWM gain과 intensity-difference squeezing을 보여줍니다. Biphoton mode는 cascade/diamond SFWM source estimate, g²SI(τ), CAR, rate, phase matching을 봅니다.

5. 왜 빠른가

OBE 계산이 빠르다는 말은 원자광학 사용자에게 꽤 큰 주장입니다. GABES가 빠른 이유는 “물리를 대충 잘라내서”가 아니라, 같은 물리를 더 계산하기 좋은 형태로 정리했기 때문입니다.

요약: GABES는 무수히 많은 작은 선형 방정식을 하나씩 반복해서 푸는 대신, 같은 구조의 Liouvillian을 한꺼번에 쌓아 풀고, Doppler/scan/field 축에서 재사용 가능한 부분을 미리 표로 만들어 둡니다. 그 결과 물리적 엄밀함을 상당히 살리면서도 interactive 사용이 가능한 속도를 얻습니다.

Velocity class를 한꺼번에 풉니다

Doppler 평균은 원래 각 속도 class마다 OBE steady state를 풀어야 해서 느립니다. GABES는 Δeff에 따른 Liouvillian 묶음을 배열로 쌓고, 같은 크기의 작은 선형연립방정식을 batched solve로 처리합니다. 사용자가 보기에는 “속도 적분”이지만, 내부적으로는 같은 꼴의 문제를 한 번에 밀어 넣는 방식입니다.

Δeff table을 재사용합니다

scan이 단순히 detuning axis를 이동시키는 경우, 매번 Hamiltonian을 새로 만들 필요가 없습니다. 예를 들어 흡수/OD 계열에서는 fine Δeff table을 만든 뒤 Maxwell weight로 평균하거나 interpolation합니다. 온도는 weight를, detuning은 축 offset을 바꾸는 쪽으로 분리됩니다.

FWM에서는 χ matrix table을 분리합니다

seeded FWM 계산에서는 probe/conjugate propagation에 필요한 χ̄ matrix를 detuning과 Δeff 격자 위에서 먼저 계산합니다. 이 table은 온도와 OPD 변화에서 일부 재사용 가능한 구조를 가지며, 이후 Maxwell-Bloch propagation과 readout이 가벼워집니다.

무거운 solve와 가벼운 readout을 나눕니다

UI에서 모든 slider가 같은 비용을 갖지 않습니다. 실제 OBE를 다시 풀어야 하는 knob만 cache key로 들어가고, 이미 계산된 curve 위에서 값을 읽는 knob는 즉시 갱신됩니다. 사용자는 TPD 같은 operating point를 움직이며 gain/squeezing 변화를 거의 실시간으로 볼 수 있습니다.

작은 행렬에 맞는 수치 전략을 씁니다

OBE의 Liouville space는 보통 n² 차원의 작은 dense matrix입니다. 이런 문제에서는 각 solve마다 거대한 멀티스레드 선형대수 라이브러리를 동원하는 것이 오히려 느릴 수 있습니다. GABES는 작은 행렬을 많이 푸는 workload에 맞게 BLAS threading을 제한하고, 필요할 때 독립 branch만 병렬화합니다.

중요한 해석

빠르다는 것은 “모델이 빈약하다”와 같은 말이 아닙니다. GABES는 많은 곳에서 실제 OBE, CG-branched decay, Doppler average, Zeeman manifold, Maxwell-Bloch propagation을 사용합니다. 다만 실험의 모든 non-ideal effect를 처음부터 풀지 않고, 우선순위가 높은 물리만 계산 가능한 형태로 잘 접은 것입니다.

6. 권장 사용 흐름

실험 조건을 처음 잡을 때

  1. 가장 가까운 scheme을 선택합니다. 예: SAS 확인이면 OD / SAS, FWM squeezing이면 FWM → Squeezing.
  2. 기본 preset을 먼저 적용합니다. 이 단계에서는 숫자 하나하나를 믿기보다 line shape과 scale을 봅니다.
  3. 온도, cell length, detuning, power처럼 실험에서 실제로 움직일 수 있는 knob만 먼저 조정합니다.
  4. metric과 plot을 함께 봅니다. 예를 들어 gain이 커졌는데 squeezing이 나빠지면 detection η나 loss knob도 확인합니다.
  5. 마지막에 Advanced / numerics를 열어 scan resolution, velocity step, relaxation rate 민감도를 확인합니다.

데이터와 비교할 때

  • absolute amplitude보다 먼저 center, width, sign, branch, scaling trend를 비교합니다.
  • 실험에 있는 loss, detector efficiency, residual field, buffer gas, transit relaxation을 가능한 범위에서 맞춥니다.
  • 계산이 실험을 설명하지 못하면, “코드가 틀렸다”보다 먼저 모델에 빠진 물리를 의심합니다.

새 scheme을 추가할 때

새 물리 구도는 Scheme을 상속하여 param_schema(), compute(params), observables(raw, params)를 정의하고 registry에 등록하는 구조입니다. UI를 직접 뜯지 않아도 control과 plot이 따라옵니다.

7. 해석상의 주의

GABES는 의도적으로 베타 toy-model solver입니다. 결과를 해석할 때는 아래 한계를 기억하는 것이 좋습니다.

영역 현재 모델의 성격 주의점
OD / SAS Hyperfine manifold, CG-branched decay, transit relaxation을 포함한 OBE 기반 모델 Ne pressure shift, Dicke narrowing 등은 단순화되어 있습니다.
Λ coherence Scalar D-line medium 위의 Λ engine 실제 polarization/Zeeman 세부구조까지 자동 포함하는 full model은 아닙니다.
Magneto-optics 87Rb D1 Zeeman manifold와 light/dark 또는 buffer relaxation 모델 cell wall, residual field, optical pumping의 실제 복잡도는 실험별 조정이 필요합니다.
Seeded FWM 85Rb D1 double-Λ gain/squeezing, regression-anchored reference model propagation loss with Langevin noise, seed excess noise, pump scattering 등은 아직 완전 모델이 아닙니다.
Biphoton SFWM reference-calibrated source estimate full quantum-Langevin propagation/noise calculation으로 해석하면 안 됩니다.
권장 태도: 숫자는 “실험 설계의 방향”으로 쓰고, 절대값은 항상 해당 scheme의 가정과 함께 읽으십시오. GABES는 실험자의 사고 속도를 올리는 도구이지, 실험적 판단을 대체하는 도구는 아닙니다.

8. 베타와 발전 방향

현재 버전은 베타입니다. 여러 scheme이 이미 동작하지만, 각 물리 영역마다 검증 수준과 모델 완성도가 다릅니다. 특히 FWM과 biphoton 쪽은 사용자가 관심을 가질 만한 기능이 빠르게 늘어날 수 있는 영역입니다.

예정 또는 자연스러운 확장 방향

  • FWM propagation에서 longitudinal phase mismatch Δkz의 더 명시적 반영
  • loss/noise를 scalar efficiency가 아니라 Langevin noise에 가까운 형태로 분리
  • pump scattering, seed excess noise, residual carrier/sideband 같은 실험적 noise source 추가
  • Raman gain, slow-light/group-index readout, higher-order wave mixing, Bell-Bloom magnetometry
  • STIRAP, Ramsey, Mollow, g²(τ) 같은 time-domain 또는 two-time correlation engine

따라서 지금은 “완성품”이라기보다, 원자광학 연구자가 실제 실험 질문을 넣어가며 성장시키기 좋은 계산 플랫폼으로 보는 것이 적절합니다.

9. 짧은 FAQ

Q. GABES는 toy model인가요?

네. 하지만 “아무 물리 없는 장난감”이라는 뜻은 아닙니다. 실험적으로 중요한 OBE 구조를 상당히 유지하되, 모든 실험 비이상성을 다 넣지는 않는다는 뜻에 가깝습니다.

Q. 왜 어떤 slider는 바로 반응하고, 어떤 slider는 오래 걸리나요?

OBE를 다시 풀어야 하는 knob와, 이미 푼 결과에서 readout 위치만 바꾸는 knob가 다르기 때문입니다. 예를 들어 FWM의 TPD는 curve navigation에 가깝지만, temperature나 OPD는 재계산이 필요합니다.

Q. 계산이 실험과 다르면 어떻게 해야 하나요?

먼저 detuning convention, transition label, beam waist의 radius/diameter convention, cell length, temperature, loss, detector efficiency를 확인하십시오. 그 다음 residual field, optical pumping, pressure broadening, phase mismatch, excess noise처럼 현재 모델 밖에 있거나 단순화된 항목을 의심하는 것이 좋습니다.

Q. 이걸 논문 figure용으로 바로 써도 되나요?

가능은 하지만, 사용한 scheme의 가정과 한계를 명확히 써야 합니다. 특히 calibrated estimate나 scalar loss knob가 들어간 결과는 first-principles 계산처럼 보이지 않게 설명해야 합니다.
